# 💻 项目方案书：跨平台防偷窥隐私调光插件 (PrivacyDimmer)

## 1. 项目背景与目标
**问题场景**：在开放式办公区、咖啡厅或通勤途中，用户在使用微信、Slack 等通讯软件，或查看机密文档时，容易被身旁的人“视觉偷窥”（Shoulder Surfing）。手动调节显示器亮度过于繁琐且容易遗忘。
**产品目标**：开发一款轻量级的桌面常驻工具，能够**自动识别**当前用户正在使用的软件。当切换到敏感软件时，屏幕自动变暗（覆盖黑色半透明层）；切换到普通软件时，屏幕立刻恢复正常亮度。

## 2. 核心功能规划 (Features)

### 2.1 MVP 阶段（最小可行性产品）
* **应用监控与识别**：实时检测当前处于前台活跃状态的应用（Active Window）。
* **动态遮罩层**：在全屏幕覆盖一层纯黑、半透明、鼠标可穿透的无边框窗口。
* **自定义敏感名单**：用户可以添加/移除触发隐私模式的应用程序（如 `WeChat.exe`, `Slack.app`）。
* **透明度无极调节**：提供滑块，允许用户自定义遮罩层的暗度（0% - 90%）。
* **全局一键开关**：支持通过主界面按钮或系统托盘（Tray）快速开启/关闭防偷窥监控。

### 2.2 进阶功能（V2.0 规划）
* **多显示器支持**：智能判断当前活跃应用在哪个屏幕，仅降低该屏幕的亮度，其他屏幕保持明亮。
* **智能游戏模式**：检测到全屏游戏运行时，自动暂停后台监控以节省性能。

## 3. 技术栈选型 (Technology Stack)

出于对桌面常驻工具**“低内存占用、包体积小、启动快”**的严苛要求，放弃 Electron，采用 **Tauri** 架构。

* **应用框架**: `Tauri` (利用系统原生 Webview 渲染，打包体积通常在 5-10MB，内存占用极低)。
* **底层逻辑与系统交互**: `Rust` 
  * 负责调用 Windows/macOS 原生 API。
  * 核心依赖库：`active-win-pos-rs` (获取活跃窗口)、`tauri-plugin-store` (本地配置持久化)。
* **前端用户界面 (UI)**: `Vue 3` (或 `React`) + `Vite` + `Tailwind CSS`。
  * 负责绘制设置面板和托盘菜单。

## 4. 架构设计与核心实现机制

### 4.1 双窗口架构 (Dual-Window System)
程序启动后，实际上维护着两个窗口：
1. **控制台窗口 (Main Window)**：包含所有设置项的常规 UI 窗口。关闭时隐藏至托盘区。
2. **遮罩层窗口 (Overlay Window)**：
   * **属性设置**：无边框 (`decorations: false`)、透明背景 (`transparent: true`)、置顶 (`always_on_top: true`)。
   * **点击穿透核心**：利用 Rust 针对双平台注入特殊指令，使该窗口完全忽略鼠标和键盘事件，实现“看得见、摸不着”的效果。
     * *Windows*: 注入 `WS_EX_TRANSPARENT` 和 `WS_EX_LAYERED` 句柄。
     * *macOS*: 调用 `ignoresMouseEvents: YES`。

### 4.2 状态监听引擎 (The Observer)
* 在 Rust 后端开启一个异步线程 (`tokio`)。
* 设定一个低频轮询（如每 500ms），调用系统 API 查询 `GetForegroundWindow`。
* 拿到当前活跃进程名后，与本地存储的“敏感名单”进行字符串比对。

### 4.3 IPC 通信与渲染 (Data Flow)
* **匹配成功**：Rust 通过 Tauri 的 IPC 事件通知 Overlay 窗口：`show()` 并设置 `opacity` 为用户预设值。
* **匹配失败**：通知 Overlay 窗口：`hide()` 或 `opacity = 0`。

## 5. 难点与风险评估 (Gotchas)

| 潜在风险 | 影响程度 | 应对方案 |
| :--- | :--- | :--- |
| **性能消耗** | 高 | 高频轮询获取前台窗口可能导致 CPU 占用升高。必须控制轮询间隔，或在可能的情况下使用系统事件钩子 (Event Hooks) 替代死循环。 |
| **macOS 权限壁垒** | 高 | 获取活跃应用名称和跨窗口遮罩，需要 macOS 的“辅助功能 (Accessibility)”权限。需要在代码中加入权限状态检测，若无权限则弹窗引导用户跳转至系统设置开启。 |
| **Windows 托盘闪烁** | 低 | 某些全屏应用会强制压制置顶窗口。需确保 Overlay 窗口的层级优先级设定为最高 (Topmost)。 |

## 6. 阶段性开发路线图 (Roadmap)

**Phase 1: 原型验证 (PoC)**
* 搭建 Tauri + Vue 骨架。
* 实现一个可以调整透明度、且鼠标能穿透过去点击下方图标的透明遮罩窗口。

**Phase 2: 系统级交互**
* 引入 Rust crate，成功在终端里持续打印出当前用户正在使用的 App 名称。
* 编写权限请求逻辑（特别是 macOS）。

**Phase 3: 业务逻辑缝合**
* 前端实现白名单的增删改查，并保存到本地 JSON。
* Rust 读取白名单，实现切换应用时的“亮/暗”自动联动。

**Phase 4: 打包与分发**
* 设计应用 Icon 和托盘菜单。
* 配置 GitHub Actions 进行 Windows `.exe` 和 macOS `.dmg` 的跨平台自动化构建发布。
